# 5ch（旧2ちゃんねる）アスキーアート（AA）の技術・文化調査

## AA職人の制作技術

### MS Pゴシック前提の文字幅設計

2ちゃんねる系掲示板のAAは、以下の表示環境を前提に設計されている。

- **書体**: MS Pゴシック（プロポーショナルフォント）
- **文字サイズ**: 16px（12pt）
- **行間**: 2px

MS Pゴシックはプロポーショナルフォントであるため、文字ごとに幅が異なる。等幅フォント（MS ゴシック）では全角文字が均一幅だが、MS Pゴシックでは例えば「し」は標準幅の約76%、「い」は約94%となる。AA職人はこの文字ごとのピクセル幅を把握した上で、上下の行で線が繋がるようにフォントを配置する。

### 文字幅の具体的な計算

AA職人「いのっく」の証言によれば、フォントの横幅は半角3dot〜全角16dotの範囲にあり、これらを計算して組み合わせることで立体感や曲線を表現する。1ドット単位の調整が求められる精密な作業である。

### 全角・半角の使い分け

| 分類 | 文字幅 | 用途 |
|------|--------|------|
| 全角文字（漢字・かな） | 約16dot | 面の塗り、密度の高い部分 |
| 半角文字（英数・半角カナ） | 約8dot（全角の半分） | 細い線、微調整 |
| 半角スペース | 約4dot | 微細な位置調整 |
| 全角スペース | 約8dot | 大きな空白 |

### 制作プロセス

1. **目から描き始める**: 顔AAでは目が最重要パーツ。瞼の輪郭、瞳のハイライト、虹彩を意識して文字を配置する
2. **曲線の表現**: 右下がりの斜線（「\」方向）は文字の選択肢が多いが、右上がりの斜線（「/」方向）は適切な文字が少なく、初心者の最初の壁となる
3. **省略と補完**: 細部は「省ける部分はとにかく省く」。人間の視覚が自然に補完する性質を利用する
4. **反復調整**: 初稿に約2時間、その後細部の調整を繰り返す。花などの複雑なパーツは4回以上描き直すこともある

### 部位別テクニック

**目の表現**:
- 「心」の字は丸みが瞳のカーブにフィットしやすい
- 「斥」「乍」をまつ毛・瞼の表現に使用
- AA職人間で蓄積された「目の描き方」のノウハウ体系がある

**髪の表現**:
- 基本は「：」でスクリーントーン的な効果を出す
- 「：：：」と「：．：」で明暗のグラデーションを表現
- より濃い部分には「州」「洲」など画数の多い漢字を使用

**手・指の表現**:
- 角度によって使える文字が限られるため、元絵のポーズ自体をAA向きに調整することがある

## AA制作ツール

### （´д｀）Edit

最も広く使われているAA制作専用エディタ。タブ切り替え式で複数のAAを同時編集可能。

**主要機能**:
- **画像トレース**: 背景に画像を投影し、上からなぞるようにAAを制作
- **テンプレート**: モナー、ギコ猫、しぃ、忍者、クックルなどの定番キャラをプリセット登録
- **半角/全角スペース可視化**: 不可視文字を色分け表示
- **ブラシツール**: スペースを指定文字で塗るブラシ、直接入力のペンシル、消しゴムの3種
- **矩形選択**: 範囲を矩形で選択し、移動・コピー・反転が可能
- **プレビュー**: 掲示板での実際の表示をシミュレート
- **出力**: BMP/GIF/JPG画像、HTML変換に対応

### Ascii Art Editor

（´д｀）Edit以前の標準ツール。Windows 98〜XP対応。

**主要機能**:
- **ピクチャートレーサー**: エディタの下に画像を表示してトレースを補助
- **オートトレーサー**: 画像をAAに自動変換（精度は限定的）
- **リアルタイム2chシミュレータ**: 掲示板表示をリアルタイムで確認
- **アートクリッパー**: 描画内容の一時保管
- **最適化機能**: 半角ダブルスペース抑制、行頭スペース抑制、行末スペース自動削除

### その他のツール

| ツール名 | 種別 | 特徴 |
|----------|------|------|
| アスキーアートメーカー | Webツール | ブラウザ上でマウスドラッグによるAA作成 |
| 文字絵エディター | デスクトップ | 画像からAAへの自動変換 |
| AA変換（ツールタロウ） | Webツール | 画像アップロードで即座にAA生成、幅・コントラスト調整可 |

### 自動変換 vs 手動制作

自動変換ツールはボタン一つで画像をAA化できるが、顔の細部などでは品質が低い。幾何学的な図形には適するが、美少女キャラなどの精密なAAは手動制作が必須。プロのAA職人は全て手動で5〜6時間かけて1作品を仕上げる。

### AA長編の発展

## トレースAA vs オリジナルAA

### トレースAA

既存のイラスト・写真・アニメの画像を元に、AAエディタの画像トレース機能を使って文字で再現する手法。

**特徴**:
- 元絵の構図・ポーズを忠実に再現
- 画像トレース機能で背景に元絵を投影しながら制作
- 精密な再現には高い技術が必要（自動変換では品質不足）
- 「トレパク（トレースパクリ）」という批判が一時期あった

**トレース論の転換**: AA職人いのっくは、フェルメールがカメラ・オブスクラ（光学装置）で映った像をトレースして絵を描いたとされる説を引き、「トレースは絵を描くための一工程・補助道具」と再定義した。元絵を機械的になぞるのではなく、文字という制約の中で表現を再構築する作業であるため、単なるコピーとは本質的に異なる。

### オリジナルAA

元絵を参照せず、AA職人が独自にデザインするAA。顔文字の拡張から発展したキャラクター（モナー、ギコ猫等）が代表例。完全な創作であるため自由度が高い反面、文字の制約の中でゼロから造形する難しさがある。

### 両者の関係

現在のAA文化では、トレースAAとオリジナルAAは対立概念ではなく、技法の違いとして共存している。やる夫スレでは既存キャラ（オリジナルAA）を物語に組み込みつつ、新規場面ではトレースAAで表現力を補う、という併用が一般的。

## AAが崩れる問題

### 崩れの原因

AAはMS Pゴシック 16px・行間2pxという特定の表示環境に完全に依存しているため、以下の条件で崩れる。

| 原因 | 詳細 |
|------|------|
| フォント不一致 | MS Pゴシック以外のフォントでは文字幅が異なり、配置がずれる |
| OS差異 | macOS・LinuxにはMS Pゴシックが標準搭載されていない |
| ブラウザ差異 | ブラウザのフォントレンダリングエンジンの違いで表示が変わる |
| フォント更新 | MS Pゴシック自体も一部字体が変更されており、古いAAが正常表示できないケースがある |
| 文字サイズ変更 | ユーザーがブラウザの文字サイズを変更すると崩れる |
| CSSフォント指定 | サイト側がフォントを指定していないと、OSデフォルトフォントで表示される |

### 対策

**CSS側の対策（サイト運営者向け）**:
```css
font-family: 'MS Pゴシック', 'MS PGothic', RobotoJAA, Monapo, 'IPA モナー Pゴシック', '梅Pゴシック', Textar, sans-serif;
```
フォントの指定順序が重要で、環境によっては全角表記「ＭＳ Ｐゴシック」と半角表記「MS PGothic」の両方を記述する必要がある。

**代替フォント（非Windows環境向け）**:
- **Monapo / Mona**: 日本語AA専用フォント
- **IPA モナー Pゴシック**: IPA フォントベースのAA対応版
- **梅Pゴシック（Ume P Gothic）**: Linux向け代替
- **RobotoJAA**: クロスプラットフォーム対応
- **Textar**: iOS/Android向けWebフォント

**ユーザー側の対策**:
- ブラウザのユーザースタイルシートでAA表示用フォントを強制指定
- 専用ブラウザ（Jane Style等）の使用（フォント設定が最適化済み）

### AAと顔文字の違い

| 項目 | 顔文字 | AA（アスキーアート） |
|------|--------|---------------------|
| 行数 | 1行 | 複数行 |
| 表現対象 | 表情・感情 | キャラクター・風景・文字装飾 |
| フォント依存 | 低い（1行なので崩れにくい） | 高い（複数行の位置合わせが必要） |
| 例 | `(´・ω・`)` `(ﾟДﾟ)` | モナー、ギコ猫などの全身AA |
| 制作難易度 | 低〜中 | 中〜極めて高い |

### AA文化の現在と未来

AA職人いのっくは、AA文化の衰退を自然な流れと認識しつつも、希少性による新たな価値を提唱している。名刺やポスターへの活用、現代アートとしての展開を提案し、AAを「現代の枯山水」と表現した。色彩を持たず文字のみで構成されるAAは、白黒写真のように「形に注目がいく」特性を持ち、制約の中での芸術性に価値がある、という見解である。
